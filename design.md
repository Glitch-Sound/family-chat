# 家族用チャット

## 該当

入院中の母も含んだ家族チャットを構築したい。
シンプルな機能、簡単な操作、無料サーバでの構築基本とする。
対応端末はPC、スマホ、タブレットとする。

なお、 **母は入院中のため、 極力簡単な操作で実現できることを最優先** とすること。

利用者は家族のみとし、最大で4名となる。
1日のアクセスも100件いけばよい方となる。

## 構成

以下構成とする

- フロントエンド
- 認証
- DB

### フロントエンド

Vue3 にて構築し、ソースは Github に登録する。
そのまま GitHub Pages にデプロイする。

### 認証

Firebase Authentication にて実現する。
メールアドレスとパスワードでの認証とする。
認証済み家族のみ read/write を許可する。

### DB

Firebase Realtime Database にて実現する。
保存内容は以下とする。

- uid
- displayName
- text
- createdAt
- updatedAt

なお、 createdAt / updatedAt は UTCのUNIXミリ秒（number）で保存し、表示時に日本時間へ変換する。

## チャット仕様

チャットはテキストのみ登録可能とし、画像などメディアの登録は不可とする。
画面は以下4つを用意し、SPA 方式で構築する。
URL ルーティングは持たなくてよい。

- ログイン画面
- 表示画面
- 追加画面
- 編集画面

### ログイン画面

メールアドレスとパスワードを入力してログインする。
Firebase Auth のログインセッション永続化機能を利用し、再入力の手間をなくすこと。

### 表示画面

以下情報を画面へ表示できること。
新しく登録された createdAt の新しい順でコメントを20件分を表示すること。

- 名前
- 日時
- コメント

名前は Firebase Authentication から取得する。

日時のタイムゾーンは日本とする。
また、表示形式 YYYY/MM/DD HH:mm とする。

表示画面の右上には追加ボタンを用意すること。
ボタンは+アイコンとすること。

各コメントの右横には編集ボタンを用意すること。
ボタンは鉛筆アイコンとすること。

### 追加画面

コメントのみ入力可能とし、登録ボタンにより反映する。
戻るボタンにて表示画面へ戻る。

### 編集画面

コメントのみ入力可能とし、登録ボタンにより反映する。
既に登録済みのコメントを入力済みの状態とすること。
編集は誰でも全員のコメントを編集可能とする。
誤操作を抑止するため、削除機能はなしでよい。
戻るボタンにて表示画面へ戻る。

## その他

### 環境変数

GitHub Pages + Firebase の環境変数管理にて実現すること。
以下手順を想定する。

1. .env.example をコミット(キー名のみ)
2. 実値は GitHub Actions Secrets に保存
3. Actions でビルド時に .env.production を生成してデプロイ

### ユーザ管理

事前に管理者が Firebase Authentication にて作成する。
表示名も事前に登録しておく。

## 手順

- [x] GitHubで本リポジトリを作成し、ローカルにクローンする。
- [x] Vue3プロジェクトを作成する（例: Vite + Vue）。
- [ ] Firebaseプロジェクトを作成する。
- [ ] Firebase Authentication を有効化し、メール/パスワード認証をONにする。
- [ ] 家族4名分のユーザーを Firebase Authentication に事前作成し、各ユーザーの表示名（displayName）を設定する。
- [ ] Firebase Realtime Database を作成する。
- [ ] Realtime Database の Security Rules を設定し、認証済みユーザーのみ read/write を許可する。
- [x] チャットデータの保存形式を実装する（`uid`, `displayName`, `text`, `createdAt`, `updatedAt`）。
- [x] ログイン画面を実装する（メール/パスワード入力、ログイン、Authセッション永続化）。
- [x] 表示画面を実装する（最新20件を `createdAt` 降順で表示、名前・日時・コメント表示、追加/編集ボタン表示）。
- [x] 追加画面を実装する（コメント入力、登録、戻る）。
- [x] 編集画面を実装する（既存コメントの初期表示、更新登録、戻る、全員のコメント編集可）。
- [x] 日時表示を実装する（UTCミリ秒を日本時間 `YYYY/MM/DD HH:mm` で表示）。
- [ ] `.env.example` を作成し、Firebase設定キー名のみをコミットする。
- [ ] GitHub Actions Secrets に Firebase 設定の実値を登録する。
- [x] GitHub Actions でビルド時に `.env.production` を生成して GitHub Pages にデプロイするワークフローを作成する。
- [ ] GitHub Pages を有効化し、Actionsデプロイを公開設定にする。
- [ ] 実機確認を行う（PC/スマホ/タブレットでログイン、投稿、編集、表示件数、日時表示を確認）。
- [ ] 家族ユーザーで受け入れ確認を行い、運用開始する。
